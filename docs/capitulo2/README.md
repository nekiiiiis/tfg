# Capítulo 2 — Disciplina de requisitos

## ¿Qué?

Segundo capítulo del TFG: busca dirigir el desarrollo hacia el sistema correcto describiendo los conceptos más importantes del contexto como objetos del dominio y los enlaces entre ellos, así como los requisitos del sistema de tal forma que se alcance el acuerdo entre el cliente, usuarios y desarrolladores sobre lo que el sistema debería hacer.

## ¿Por qué?

Habiendo entendido el escenario, el estado del arte y la metodología, toca dejar definidos los límites de la solución a desarrollar.

## ¿Para qué?

- Comprender la estructura y dinámica de la organización en la que el sistema se desarrollará
- Definir los límites del sistema
- Proveer una mejor comprensión de los requisitos del sistema

## ¿Cómo?

Realizando una abstracción de la realidad y a partir de esta especificando los requisitos del sistema.
